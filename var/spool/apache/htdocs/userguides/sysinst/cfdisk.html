• Before The System Can Be Installed The Hard Drive(s) Needs To Be Prepared Or Partitioned
  • This Involves Allocating Areas Of The Disk For Swap Space (Virtual Memory) And Data Storage
  • We Recommend Allocating At Least One 128M Partition To Virtual Memory
  • The Data Partitions Will Depend On The Hard Drive(s) And Requirements Of The System
  • There Is No Substantial Benefit In Creating Many Partitions On A Single Disk
  • Some Systems Are Not Able To Address Large Hard Drives In This Case Create A 64Mb Data Partition To Be Mounted As /boot Latter
  • Here Is A List Of Data Areas You May Want To Partition
    • / This Is The Beginning Of Your File System (Root Directory) And Has To Exist And Be Marked Bootable Unless A /boot Partition Is Defined
    • /boot This Is Where The Kernel Is Stored And Must Be Marked Bootable If Defined
    • /var This Is Where All The Data Is Stored IE Mail Boxes, Incoming/Outgoing Mail, File Server And Proxy Server And Requires As Much Space As Possible
    • /var/home User Home Directories And Profiles It Is A Good Idea To Have A Alternate Drive For This Data Area
    • /var/spool/samba This Is The Location Of The File Server Once Again It Is Good Practice To Store This On A Separate Disk
    • /var/spool/squid This Is Where Cached Files Are Stored And Should Be 20% Bigger Than The Size Assigned For This Purpose In The Management Console
    • /var/spool/apache All Files Associated For The Web Server
    • /var/spool/mail All Users Mail Is Stored In This Area , Required Size Depends On Number Of Users And Mail Policy
  • Once You Have Selected The Fdisk Option You Will Need To Select A Disk To Partition (This Must Be Repeated For Each Disk)
  • The Following Table Maps Familiar Drives To Linux Devices
  • If Your SCSI Drives Are Not Shown Load The Appropriate Driver (See The Drivers Section)
    • Primary Master /dev/hda
    • Primary Slave /dev/hdb
    • Secondary Master /dev/hdc
    • Secondary Slave /dev/hdd
  • If This Is The First Time You Configuring The System Select Yes
  • If You Select No You Will Be Able To Edit The Partition Table
  • The Fdisk Program Is Really Straight Forward And Uncomplicated Showing You The Disk Information And Partitions
  • You Will Create New Partions On The Free Space (Highlighted) Write The Partitions And Quit The Program
  • By Selecting New You Are Given A Choice Of Options When Possible Only Use Primary Partitions (Only 4 Per Disk Available) A Logical Partition Allows You To Create Extended Partitions Within The Space Allocated To The Logical Partition
  • Enter The Size To Create In This Case 128 Mb To Be Used As Swap Space
  • We Are Able To Create Partitions At The Beginning Or The End Of The Free Space It Is Recommended To Always Use The Beginning
  • On Linux Systems We Use The Linux Partition Type There Are Many Other Types As You Will See. When We Create Swap Swap Space The Type Needs To Change To Linux Swap
  • Highlight The Partition To Change Select Type A List Of Types Will Be Displayed
  • Press Any Key To Continue
  • We Are Only Interested With Types 82,83,85
  • Enter The Type Required In This Case 82
  • Once Successfully Added Select Free Space To Allocate The Next Partition Repeat This Process Till All Required Space Is Allocated
  • Specify The Size To Allocate For New Partition
  • If You Are Not Going To Allocate A /boot Partition Mark The / Partition Bootable By Selecting Bootable
  • Once All Required Partitions Are Allocated You Will Have To Activate The Partition Table By Selecting Write
  • You Will Be Warned About Data Loss And Prompted For A Reply Type In "yes" And Press Enter
  • All That Is Left To Do Is Quit And Start The Process Over For The Next Disk Until All Disks Are Partitioned
